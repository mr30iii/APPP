Introduction to Ethical Hacking

Ethical hacking involves an authorized attempt to gain unauthorized access to a computer system, application, or data. Carrying out an ethical hack involves duplicating strategies and actions of malicious attackers.

This practice helps to identify security vulnerabilities which can then be resolved before a malicious attacker has the opportunity to exploit them.

Key Terminology

# Common hacking terminology
Vulnerability - A weakness in a system that can be exploited
Exploit - A piece of code that takes advantage of a vulnerability
Payload - The part of an exploit that performs the desired action
Zero-day - A vulnerability that is unknown to the vendor

Ethical Hacking Principles

Ethical hackers must follow important guidelines:

  • Obtain proper authorization before accessing systems
  • Define the scope of the testing clearly
  • Report all vulnerabilities discovered
  • Respect the privacy of the organization
  • Securely delete all data obtained during testing
